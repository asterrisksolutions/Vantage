## Project Overview

**VANTAGE** is a role-based document management system built with Laravel. It supports three user roles:

- **Admin** - Full system access, manages users and roles
- **Manager** - Limited administrative access, manages teams and reports
- **User** - Standard account with basic application access

Key features:
- Session-based authentication with account lockout protection
- Role-based route protection via middleware
- Login attempt tracking (`failed_attempts`)
- Account lock status (`is_locked`)
- Last login timestamp tracking (`last_login_at`)
- SQLite database (zero-config, works out of the box)

## Installation Steps

### 1. Clone the Repository

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO/php-laravel
```

> **Note:** All commands from this point forward assume you are in the `php-laravel/` directory.

---

### 2. Install PHP Dependencies

```bash
composer install
```

This installs Laravel Framework, Sanctum (API authentication), Tinker (debugging console), and all other PHP packages.

---

### 3. Install Node.js Dependencies

```bash
npm install
```

This installs Vite (build tool), Tailwind CSS (styling), and Laravel's Vite plugin.

---

### 4. Build Frontend Assets

```bash
npm run build
```

Compiles CSS and JavaScript for production. Run this at least once before serving.

## Environment Configuration

### 1. Create the Environment File

```bash
cp .env.example .env
```

> If `.env.example` does not exist, create `.env` manually with the content below.

### 2. Generate Application Key

```bash
php artisan key:generate
```

This creates a secure 32-character encryption key used for session/cookie encryption.

### 3. Required `.env` Variables

Open `.env` and ensure these values are set:

```env
# Application basics
APP_NAME=VANTAGE
APP_ENV=local
APP_KEY=          # Auto-generated by key:generate command
APP_DEBUG=true
APP_URL=http://localhost:8000

# Database (SQLite - zero configuration, no MySQL/Postgres needed)
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite

# Session (file-based, included with Laravel)
SESSION_DRIVER=file
SESSION_LIFETIME=120

# Cache (file-based)
CACHE_STORE=file
```

> **Important:** The project uses **SQLite** by default. No separate database server (MySQL, Postgres) is required. The database file is created automatically.

## Database Setup

### 1. Create the SQLite Database File

```bash
touch database/database.sqlite
```

> On Windows PowerShell: `New-Item -ItemType File -Path database/database.sqlite`
> On Windows CMD: `type nul > database\database.sqlite`

### 2. Run Migrations and Seeders

```bash
php artisan migrate:fresh --seed
```

This command:
- Creates all tables (`users`, `roles`, `cache`, `jobs`, `sessions`, etc.)
- Seeds the database with 3 test roles (Admin, Manager, User)
- Seeds the database with 3 test user accounts

> **Note:** If you omit `--seed`, the database will be empty and no login accounts will exist.

### 3. Verify Database

Optional - check the database contents:

```bash
php artisan tinker
>>> User::all();
>>> Role::all();
```

## Running the Application

### Development Server

```bash
php artisan serve
```

The application will be available at: **http://127.0.0.1:8000**

### Stop the Server

Press `Ctrl + C` in the terminal to stop.

### Hot Reload Development (Optional)

If you plan to edit frontend assets (CSS/JS):

```bash
# Terminal 1
php artisan serve

# Terminal 2
npm run dev
```

`npm run dev` starts Vite in development mode for instant CSS/JS updates.

## Troubleshooting

### Issue: "SQLSTATE[HY000]: General error: 1 no such table" or migrations fail

**Cause:** SQLite database file does not exist or migrations were not run.

**Fix:**
```bash
touch database/database.sqlite
php artisan migrate:fresh --seed
```

---

### Issue: "User not existing or incorrect credentials" error on login

**Cause 1:** Database was seeded while `DatabaseSeeder` was double-hashing passwords (fixed in current version).

**Fix:**
```bash
php artisan migrate:fresh --seed
```

**Cause 2:** Wrong credentials. Verify by checking `database/seeders/DatabaseSeeder.php` or `TEST_ACCOUNTS.md`.

---

### Issue: "The page has expired due to inactivity" (419 error)

**Cause:** CSRF token mismatch.

**Fix:** Clear browser cookies/cache for `127.0.0.1:8000` and refresh.

---

### Issue: "Whoops, looks like something went wrong" (500 error)

**Cause:** Application encryption key missing or `.env` file missing.

**Fix:**
```bash
cp .env.example .env
php artisan key:generate
php artisan config:clear
```

---

### Issue: CSS styles not loading

**Cause:** Frontend assets not built.

**Fix:**
```bash
npm install
npm run build
```

---

### Issue: "Command not found: php artisan"

**Cause:** Not in the `php-laravel` directory, or PHP is not in system PATH.

**Fix:**
```bash
cd php-laravel
# Or use full path to PHP:
/path/to/php.exe artisan serve
```

---

### Issue: Port 8000 already in use

**Fix:** Use a different port:
```bash
php artisan serve --port=8080
```
